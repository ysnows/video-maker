# Seedance 2.0 平台参数与限制

> 基于即梦 Seedance 2.0 官方文档整理。

## 输入规格

| 维度 | 规格 |
|------|------|
| **图片输入** | jpeg/png/webp/bmp/tiff/gif，≤9张，单张 <30MB |
| **视频输入** | mp4/mov，≤3个，总时长 2-15秒，单个 <50MB |
| **视频像素范围** | 480p(640x640) ~ 720p(834x1112)，即总像素数 409,600 ~ 927,408 |
| **音频输入** | mp3/wav，≤3个，总时长 ≤15秒，单个 <15MB |
| **文本输入** | 自然语言描述 |
| **混合上限** | 最多 12 个文件（图片 + 视频 + 音频合计） |

**素材优先级**：建议优先上传对画面或节奏影响最大的素材，合理分配不同模态的文件数量。

## 输出规格

| 维度 | 规格 |
|------|------|
| **生成时长** | 4-15秒，可自由选择 |
| **分辨率** | 支持 2K 输出 |
| **声音** | 自带音效/配乐生成 |

## 两种入口

1. **首尾帧入口**：仅首帧图 + 提示词（简单场景）
   - 智能多帧和主体参考在此入口下无法选中
2. **全能参考入口**：多模态（图、视频、音频、文本）组合输入
   - 通过 `@素材名` 指定每个素材的用途
   - 需明确说明每个素材用途

## 平台限制

- **不支持上传含有写实真人脸部的素材**（图片和视频均不可），系统会自动拦截
- 有参考视频时生成消耗更多算力
- 视频延长时，选择的生成时长应为"新增部分"的时长（延长 5 秒，生成长度也选 5 秒）
- 单次生成最长 15 秒，超过需用分段拼接

## @引用官方命名

| 类型 | 命名 | 数量上限 |
|------|------|----------|
| 图片 | `@图片1`、`@图片2`、...、`@图片9` | 9 张 |
| 视频 | `@视频1`、`@视频2`、`@视频3` | 3 个 |
| 音频 | `@音频1`、`@音频2`、`@音频3` | 3 个 |

**注意**：不要使用 @img1、@video1、@audio1 等英文命名。

## @引用常见用法

```
@图片1为首帧
@图片1的人物形象
参考@视频1的运镜效果
参考@视频1的打斗动作
背景音乐参考@音频1
将@视频1延长5s
在@视频1和@视频2之间加一个场景，内容为xxx
```

## 官方 Tips

- **参考 vs 编辑要写清楚**：「参考」是借鉴风格/动作/运镜，「编辑」是在原素材上修改
- **没有音频素材？** 可以直接参考视频里的声音
- **想融合多个视频？** 提示词中说明合成逻辑
- **连续动作？** 加入连续性描述，如"角色从跳跃直接过渡到翻滚，保持动作连贯流畅"
- **素材多时务必检查**：各个 @对象有没有标清楚，别把图、视频、角色搞混
