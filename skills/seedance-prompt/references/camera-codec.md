# 相机四维编码系统（Camera Codec）

> 运镜是决定视频质量的关键。同样的场景，加上精准运镜描述就是"电影感"和"随手拍"的区别。
>
> **核心思维转换**：不再记忆零散关键词，而是在脑中建立坐标系 → 先定位 Z/Y/X → 再叠加 F → 最后翻译成自然语言提示词。
>
> Seedance 2.0 对中英文混合描述效果更好。

---

## 总公式

```
一个完整镜头 = [Z 距离] + [Y 高度] + [X 方位] + [F 滤镜] + [运动] + [节奏] + [约束]
      位置层                              能力层      运动层    节奏层    防漂移
```

**三层理解：**

| 层级 | 控制什么 | 对应维度 |
|:---|:---|:---|
| 位置层 | 相机放在哪里 | Z轴（远近）+ Y轴（高低）+ X轴（环绕） |
| 能力层 | 相机镜头的物理特性 | F层（焦段、景深、畸变、叙事身份） |
| 运动层 | 相机怎么动 | Z/Y/X 坐标随时间的动态变化 |

---

## 第一维度 Z轴 —— 距离与信息量

> 相机与主体的距离。决定画面是"看清毛孔"还是"看清世界"。

| 编码 | 名称 | 英文 | 画面范围 | 核心传递 | 典型用途 |
|:---|:---|:---|:---|:---|:---|
| **Z1** | 大特写 | ECU | 瞳孔、指纹、伤口 | 极致感官刺激 | 悬疑线索、情绪爆发瞬间 |
| **Z2** | 特写 | BCU | 眼睛、嘴巴、手 | 局部特征强调 | 内心独白、物品细节 |
| **Z3** | 近景 | CU | 完整面部 | 表情、情感高潮 | 对话反应镜头 |
| **Z4** | 中近景 | MCU | 胸口以上 | 上半身表达 | **最通用**：对话、Vlog |
| **Z5** | 中景 | MS | 腰部以上 | 肢体动作、交互 | 拿东西、指路、手势 |
| **Z6** | 四分之三身 | MLS | 膝盖附近 | 重心与手部关系 | 牛仔镜头、站姿 |
| **Z7** | 全景 | FS | 完整人体+环境 | 动作展示、亮相 | 穿搭、舞蹈、亮相 |
| **Z8** | 远景 | LS | 人小景大 | 环境关系 | 建立场景、交代地理 |
| **Z9** | 大远景 | ELS | 城市、山脉 | 世界观、宏大叙事 | 开场建立、史诗场景 |

**速查口诀：**

```
看清毛孔 ← Z1 Z2 Z3 → 看清表情
看清动作 ← Z4 Z5 Z6 → 看清姿态
看清世界 ← Z7 Z8 Z9 → 看清全局
```

## 第四维度 F层 —— 镜头滤镜与特殊属性

> 不是相机的位置，而是镜头的**物理能力**和**叙事身份**。叠加在 Z/Y/X 坐标之上。

### F1 物理光学（焦段）

| 焦段 | 空间感 | 虚化 | 情绪 | 适用 |
|:---|:---|:---|:---|:---|
| 18mm 超广角 | 极度夸张 | 几乎无 | 宏大、眩晕 | 建筑、极限运动 |
| 24mm 广角 | 轻微夸张 | 轻微 | 开阔、客观 | 室内全景 |
| 35mm 标准 | 接近人眼 | 中等 | 自然、叙事 | **万能**：日常纪录 |
| 50mm 标准 | 最自然 | 中偏强 | 经典、电影感 | **电影标准**：对话 |
| 85mm 长焦 | 轻微压缩 | 强 | 柔美、亲密 | **人像黄金**：特写 |
| 135mm+ | 强烈压缩 | 极强 | 窥视、戏剧 | 远距偷拍、背景压缩 |

**景深类型：**
- 浅景深 — 主体锐利，背景虚化（人像、产品）
- 深景深 — 前后景都清晰（环境交代、群像）
- 拉焦 Rack Focus — 焦点从 A 转到 B（悬念揭示）
- 散景 Bokeh — 背景光斑柔美光球（夜景、浪漫）

**光学畸变：**
- 鱼眼 — 极端桶形畸变（滑板视频、嘻哈 MV）
- 广角拉伸 — 边缘物体被拉伸（恐怖、不安）
- 变形宽银幕 anamorphic — 水平光晕、椭圆散景（电影质感）
- 微距 macro — 极近对焦，放大微小物体（昆虫、珠宝）

### F2 身份叙事

| 类型 | 提示词 | 效果 | 典型用途 |
|:---|:---|:---|:---|
| POV 主观 | "第一人称视角 / POV shot" | 代入角色眼睛 | 恐怖片、沉浸体验 |
| 过肩 OTS | "over-the-shoulder" | 两人空间关系 | 对话、谈判 |
| 偷窥 Voyeur | "透过门缝/栅栏/窗户" | 被监视感 | 悬疑、惊悚 |
| 镜面 Mirror | "通过镜面反射/水面倒影" | 双重性、自我审视 | 内心独白 |

### F3 构图几何

| 类型 | 提示词 | 效果 | 典型用途 |
|:---|:---|:---|:---|
| 荷兰角 Dutch | "倾斜构图 15°" | 不安、失衡、动荡 | 惊悚、追逐 |
| 对称 Symmetry | "完美对称居中" | 秩序、仪式、压迫 | 韦斯·安德森、宗教 |
| 引导线 Leading | "走廊/铁轨形成引导线" | 深度、方向感 | 目标展示 |

## 运镜词典（坐标变化的动态过程）

所有运动归结为六种类型：**Z推拉 / Y升降 / X横移 / 旋转 / 复合 / 光学**。

### 基础运镜（覆盖 80% 需求）

| 动作 | 英文 | 坐标变化 | 效果 |
|:---|:---|:---|:---|
| 推进 | Dolly In / Push In | Z↓ 靠近 | 增加亲密感、聚焦 |
| 拉远 | Dolly Out / Pull Out | Z↑ 远离 | 揭示环境、疏离 |
| 升起 | Crane Up / Pedestal Up | Y↑ | 揭示全景、升华 |
| 降落 | Crane Down / Pedestal Down | Y↓ | 聚焦地面、压抑 |
| 左平移 | Truck Left | X← | 展示空间 |
| 右平移 | Truck Right | X→ | 展示空间 |
| 左摇 | Pan Left | Y轴旋转 | 水平扫描、跟随 |
| 右摇 | Pan Right | Y轴旋转 | 水平扫描、跟随 |
| 上摇 | Tilt Up | X轴旋转 | 仰望揭示高度 |
| 下摇 | Tilt Down | X轴旋转 | 俯瞰地面 |
| 滚转 | Roll / Dutch Roll | Z轴旋转 | 眩晕、梦境 |
| 变焦 | Zoom In/Out | 焦段变化 | 快速放大/缩小 |
| 拉焦 | Rack Focus | 焦点切换 | 引导注意力 |
| 跟踪 | Tracking Shot | 随主体移动 | 跟随主体 |

### 高级运镜

| 动作 | 坐标行为 | 效果 | 场景 |
|:---|:---|:---|:---|
| 呼吸感推进 | Z 微幅往复 | 缓慢推进带微妙起伏 | 悬念、凝视 |

### 镜头质感修饰（隐藏层级）

在运镜动作之外叠加镜头本身的物理状态，"不完美"反而大幅提升真实感，是区分普通和大制作质感的关键：

- 雾水珠附着镜头前、雾粒粘镜
- 雨滴溅到镜头、水珠滚落
- 镜头轻微抖动、爆炸冲击波震动镜头
- 镜头畸变 + 雾层折射
- 镜头耀斑 lens flare、逆光眩光

## 运镜冲突检测（输出前自动检查）

| 冲突类型 | 规则 | 修正建议 |
|:---|:---|:---|
| **三轴同动** | Z推+X环绕+Y升降同时 | 拆成 2 个镜头，或去掉 1 个轴 |
| **近距+大旋转** | Z1-Z3 + X 轴大幅旋转 | 容易崩脸，换为 Z4+ 或减小旋转幅度 |
| **文本视频矛盾** | 提示词说"左移" + 参考视频在"右移" | 统一方向 |
| **运镜冲突** | 同时要求推进+拉远 | 选择一个，或改为希区柯克变焦 |
| **节奏冲突** | "缓慢"+"快速" 同时出现 | 明确单一节奏词 |

**核心铁律**：**每个镜头最多双轴运动。三轴同时变化 = 失控。**

## 多模态素材分工（参考视频驱动运镜）

> **参考视频控制"坐标怎么变化"（运动轨迹），文本控制"坐标是什么"（起始位置、内容、风格）。**

### 分工式多素材

```
@图片1  → 锁定角色外貌（Z3 近景时的面部细节）
@图片2  → 锁定场景色调（F:色调滤镜）
@视频1  → 锁定坐标变化轨迹（Z/Y/X 运动）
@音频1  → 锁定音频节奏（节拍与运镜同步）
```

### 提示词分工模板

```
"运镜完全参考 @视频1（环绕运镜参考）的镜头运动方式，
 人物外貌参考 @图片1（角色名），
 场景色调参考 @图片2（场景参考）。
 [Z 景别 + Y 高度 + X 方位 + F 滤镜 + 内容描述]。
 稳定流畅，真实摄影机运动。
 不要复刻 @视频1 中的人物和场景。"
```

**断句防歧义铁律**：所有 `@图N` 引用后必须紧跟角色名或名词解释。
- ✅ `@图片1（李武）站起身`
- ❌ `@图片1 跑向...`（歧义）

### 参考视频准备规范

| 要求 | 说明 | 错误 |
|:---|:---|:---|
| 时长 | 3-8 秒最佳 | 太短运动不完整 |
| 运动单一 | 一个视频只传达一种坐标变化 | 多轴同时会混乱 |
| 素材清理 | 剪掉开头调整的 1-2 秒 | 起手抖动被学习 |
| 稳定性 | 用三脚架/稳定器素材 | 抖动被忠实复刻 |
| 画质 | 高清素材 | 低码率导致闪烁 |

## 防漂移约束（自动追加）

所有输出提示词必须自带防崩坏兜底。

**运镜稳定：**
```
稳定流畅、真实摄影机运动、不要几何扭曲、运动缓入缓出
```

**角色锁定（有人物时）：**
```
保持人物身份一致、面部比例稳定、不要变脸、五官不变形
```

**空间锁定（有建筑/场景时）：**
```
背景结构稳定、直线保持笔直、建筑物不弯曲、真实透视
```

**画质锁定：**
```
4K 超高清、细节清晰、无模糊闪烁
```

**反向排除：**
```
模糊、低画质、面部变形、多余手指、肢体扭曲、闪烁、跳帧、几何扭曲、镜头抖动、画面撕裂
```

## 输出格式

### 快速模式（用户说"快速/直接给提示词"）

只输出坐标编码 + 提示词。

### 完整模式（默认）

```
━━━ 📷 镜头编码 ━━━━━━━━━━━━━━━━

▎坐标编码：[Z值] + [Y值] + [X值] + [运动] + [F:属性]

▎提示词（可直接复制）：
"[景别+角度+方位]，[运镜动作+节奏]。
 [主体+动作描述]。[场景+光影]。
 [焦段]，[景深类型]。
 稳定流畅，真实摄影机运动，缓入缓出。
 不要面部变形，不要几何扭曲。"

▎编码解析：
  Z[值] = [含义]
  Y[值] = [含义]
  X[值] = [含义]
  运动 = [含义]
  F = [含义]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 八大核心要素自检清单

输出每个镜头提示词前必须自检：

| # | 要素 | 检查问题 | 来源维度 |
|:---|:---|:---|:---|
| 1 | 精准主体 | 谁？具体到外貌特征和穿着 | 内容层 |
| 2 | 动作细节 | 在干什么？具体到肢体动作 | 内容层 |
| 3 | 场景环境 | 在哪？具体到环境元素 | 内容层 |
| 4 | 光影色调 | 什么氛围？具体到色温/光比 | F 层 |
| 5 | 镜头运镜 | 怎么拍？具体到坐标变化 | Z/Y/X + 运动层 |
| 6 | 视觉风格 | 什么画风？具体到渲染标准 | F 层 + 美学 |
| 7 | 画质参数 | 清晰度要求？4K/8K？ | 约束层 |
| 8 | 约束条件 | 防崩坏兜底？ | 约束层 |

**缺失处理**：
- 缺 1-3 个要素 → 主动向用户提问补充
- 要素完备 → 直接输出
- **拒绝静默修改**：要素缺失时必须先向用户展示建议让其选择，不得自动猜测填充。

## DO / DON'T 强制规范

### ✅ DO

- 先定坐标（Z+Y+X），再叠滤镜（F），再写运动和节奏
- 每个镜头只做单轴或双轴运动
- 用情绪词描述速度（"缓慢推近，紧张感上升"），不要用数字（"每秒 5 像素"）
- 明确写出焦段数值（不要只写"长焦"，写"85mm"）
- 提示词长度控制在 50-150 字
- 多镜头叙事用坐标递进制造节奏（Z9→Z5→Z2）
- 同一提示词生成 2-3 次择优
- `@图N / @视频N` 后面必须紧跟角色名或解释词
- 必须加稳定约束词

### ❌ DON'T

- 不要在一个镜头同时做三轴运动
- 不要在 Z1-Z3 做 X 轴大幅旋转（容易崩脸）
- 不要让提示词超过 200 字
- 不要让运动速度过快
- 不要只写"虚化"，明确写出焦段+景深类型
- 不要忽略 X 轴方位
- 不要让 Asset ID 独立出现在提示词中（必须 `@图N（含义）`）
- 不要在一个时间切片内混合多种运镜方式

## 常见问题排查

| 问题 | 涉及维度 | 可能原因 | 解决方案 |
|:---|:---|:---|:---|
| 运镜路径不一致 | 运动层 | 参考视频多轴冲突 | 换单一运动三脚架素材 |
| 画面闪烁 | F 层 | 参考视频压缩伪影 | 换高清稳定素材 |
| 每次生成差异大 | 全维度 | 参考太复杂 | 简化参考，只保留单轴 |
| 视频抖动卡顿 | 运动层 | 参考有突然变速 | 保持匀速 |
| 运镜方向反了 | X/Z 轴 | 文本和视频方向矛盾 | 统一方向 |
| 镜头突然乱飘 | 运动层 | 多轴叠加 | 单轴或双轴即可 |
| 运镜太快/太慢 | 节奏 | 节奏词不明 | 用情绪词替代数字 |
| 角色在运镜中崩脸 | Z+X | Z 太近 + X 大旋转 | 避免 Z1-Z3 + X 轴旋转 |
| 环绕时背景变形 | X+F | 没加空间约束 | 加"直线笔直，建筑不弯曲" |
| 跟拍时主体出画 | X/Z | 运动幅度过大 | 降低速度，缩小幅度 |
| 景深不稳定 | F | 没明确焦段和景深 | 写"85mm，浅景深" |
| 构图偏移 | X | 方位描述不精确 | 明确"正面/3/4 侧/侧面" |

## 与三级运镜体系的对照（新旧兼容）

四维编码系统**兼容并升级**了原先的三级运镜体系：

| 旧体系（[vocabulary.md](vocabulary.md)） | 新体系（本文件） |
|:---|:---|
| Level 1 基础动作 Pan/Dolly/Zoom/Crane/Orbit... | 运镜词典 → 基础运镜（坐标变化的动态过程） |
| Level 2 修饰词 Smooth/Slow/Cinematic... | 节奏层（坐标变化的速度与情绪） |
| Level 3 组合技 Orbit + Zoom In... | 多轴组合（受"最多双轴"铁律约束） |

**建议工作流**：
- **新手入门** → 记熟 Level 1 的 6 个基础动作（Pan/Dolly/Zoom/Crane/Orbit/Tracking），覆盖 80% 需求
- **进阶** → 切换到四维编码思维，用 Z+Y+X+F 锁定每个镜头
- **精准控制** → 分工式多素材控制 + 坐标递进多镜头叙事
