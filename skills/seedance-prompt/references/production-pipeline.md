# 长视频生产流水线（>15秒）

> 长视频不是"长的短视频"。≤15秒可以靠一条提示词搞定，>15秒必须走完整的前期制作流程。跳过任何一步都会导致成片质量断崖式下降。

---

## 完整流程

```
① 概念/剧本 → ② 角色设计 → ③ 角色卡片图 → ④ 分镜脚本 → ⑤ 分镜参考图 → ⑥ 逐镜头视频生成 → ⑦ 后期拼接
```

### ① 概念/剧本

- 一句话说清楚"这个视频讲什么"
- 明确总时长、分几段、每段的叙事任务
- 使用三段式骨架：建立世界 → 变化推进 → 情绪收束

### ② 角色设计

- 确定角色数量、各角色的外观特征（五官、发型、服装、体型、标志性道具）
- 写出角色描述文档，供后续所有环节引用
- 关键：描述要足够具体，以确保在多次生成中角色外观一致

### ③ 角色卡片图生成

- 用即梦的**图片生成**功能，为每个角色生成多角度参考图
- 推荐生成：正面、侧面、全身、半身各一张
- 这些图片将作为 @图片N 在后续视频生成中引用，是角色一致性的锚点
- **角色卡片图的质量直接决定成片中角色的一致性**

### ④ 分镜脚本

- 按时间轴拆分为具体镜头（每个镜头 ≤15秒）
- 每个镜头写明：画面内容、运镜方式、角色动作、情绪、音效/台词
- 标注镜头之间的衔接方式（桥接镜头/同构图推进/时间流逝）
- 分镜脚本 = 视频的蓝图，写得越细，后续越可控

### ⑤ 分镜参考图

- 为关键镜头生成参考图片（首帧/关键帧）
- 用即梦图片生成，提示词基于分镜脚本描述
- 参考图的作用：
  - 首帧图：锚定每段视频的起始画面
  - 关键帧图：引导模型理解特定画面构图
  - 风格参考图：统一全片视觉调性

### ⑥ 逐镜头视频生成

- 第1段：首帧图 @图片N + 角色参考 @图片N + 提示词 → 正常生成
- 第2段起：上一段视频 @视频1 + 角色参考 @图片N + 延长提示词 → 视频延长
- 每段都重复锁定语句（风格/角色/场景）
- 每段都附加负面提示词
- **不满意就重新生成该段，不要将就**——一段凑合会拖垮全片

### ⑦ 后期拼接

- 使用剪映/PR/达芬奇等工具拼接各段
- 添加转场（如果段间不够丝滑）
- 统一调色确保各段色调一致
- 添加音乐/音效/字幕
- 最终校验节奏和连贯性
- **剪辑节奏决定成片质感**：必须按明确公式剪辑，不要"凭感觉"——详见 [editing-rhythm.md](editing-rhythm.md)

## 常见问题

**Q：角色卡片图生成出来不够一致怎么办？**
A：多生成几张，挑最好的一张作为锚点。后续所有视频生成都引用同一张角色图。一致性来自"反复引用同一张参考图"，而非"希望模型自己记住角色"。

**Q：分镜参考图和首帧图有什么区别？**
A：首帧图是"视频从这张图开始动"，需要画面精确；分镜参考图是"给模型看一下这个镜头大概长什么样"，可以更概念化。

**Q：>60秒的视频怎么做？**
A：拆分为独立场景分别生成（每个场景 ≤30-45秒），最后用剪辑软件拼接。不要试图用视频延长做超过60秒的连续视频——漂移风险太高。

**Q：可以跳过某些步骤吗？**
A：角色卡片图和分镜参考图是最不能跳过的。跳过它们 = 把一致性完全交给运气。

## 进阶：25格制作流水线（长视频标准化输出）

> 当项目 **≥45 秒** 或需要**可交付的规范化方案**时，推荐使用 25 格两阶段流水线。它把上文的 ① ~ ⑥ 步骤结构化为可复用的模板。

### 两阶段流程

```
━━━ 🎬 25格制作流水线 ━━━━━━━━━━━━━━━━

┌────────────────────────────────────────────┐
│  Phase 1：25格剧情拆解                      │
│  输入：创意 / 脚本 / 一句话故事线            │
│  输出：《25格剧情拆解表》                    │
│         ├─ 格数（1-25）                     │
│         ├─ 画面内容                         │
│         ├─ 景别（Z 轴坐标编码）              │
│         ├─ 运镜建议（坐标变化）              │
│         ├─ 美学标注（色调 + 构图）           │
│         └─ 时长建议                         │
│         ❗ 等待用户确认 ❗                   │
├────────────────────────────────────────────┤
│  Phase 2：5×5 聚合提示词 / 分镜图           │
│  模式 A：5×5 聚合提示词矩阵                 │
│  模式 B：逐格高质量分镜图（P0/P1 优先）      │
│  模式 C：双模式混合（推荐）                  │
└────────────────────────────────────────────┘
```

### 第一阶段：25格剧情拆解表

```
▎项目名：[名称]
▎总时长：[预估秒数]
▎美学基调：[色调组合名] + [渲染标准]

| 格# | 时长 | 画面内容 | 景别 | 运镜 | 美学 | 叙事功能 |
|-----|------|----------|------|------|------|----------|
| 01  | 2s   | [描述]   | Z8   | Crane Down | 冷区·深空蓝 | 世界观 |
| 02  | 3s   | [描述]   | Z5   | Steadicam  | 过渡      | 角色引入 |
| ... | ...  | ...      | ...  | ...        | ...       | ...     |
| 25  | 2s   | [描述]   | Z9   | Crane Up   | 回归冷区  | 留白收尾 |
```

- **景别编码**：遵循 [camera-codec.md](camera-codec.md) 的 Z 轴系统
- **美学标注**：遵循 [aesthetic-constraints.md](aesthetic-constraints.md) 的色调组合
- **叙事功能**：世界观建立 / 角色引入 / 冲突建立 / 高潮 / 回落 / 留白

### 25格景别分配建议

```
格 01-05（建立段）：Z8 → Z5，从远到近建立世界观
格 06-10（发展段）：Z5 ↔ Z4，中景来回，动作与对话
格 11-16（升温段）：Z4 → Z3，逐步拉近，紧张感加剧
格 17-19（高潮段）：Z2 → Z1，特写/大特写，情绪爆发
格 20-23（回落段）：Z3 → Z7，拉远回落
格 24-25（留白段）：Z8 → Z9，大远景收尾
```

### 第二阶段：5×5 聚合提示词矩阵

将 25 格排列为 5×5 矩阵，每格一个独立完整的提示词：

```
┌────────┬────────┬────────┬────────┬────────┐
│ 格01   │ 格02   │ 格03   │ 格04   │ 格05   │
├────────┼────────┼────────┼────────┼────────┤
│ 格06   │ 格07   │ 格08   │ 格09   │ 格10   │
├────────┼────────┼────────┼────────┼────────┤
│ 格11   │ 格12   │ 格13   │ 格14   │ 格15   │
├────────┼────────┼────────┼────────┼────────┤
│ 格16   │ 格17   │ 格18   │ 格19   │ 格20   │
├────────┼────────┼────────┼────────┼────────┤
│ 格21   │ 格22   │ 格23   │ 格24   │ 格25   │
└────────┴────────┴────────┴────────┴────────┘

每格提示词结构：
"[景别+角度+运镜]。[主体+动作]。[场景+光影]。
 [美学约束：色调+材质+构图]。[防崩坏约束]。"
```

**分镜图生成优先级**（用 `image_gen` 辅助）：

- **P0（必须生成）**：高潮帧 17-19 + 开场帧 01 + 结尾帧 25
- **P1（推荐生成）**：转折帧 06、11、16、20
- **P2（可选生成）**：其余补充帧

### 适配注意事项

```
✅ 每格提示词独立完整，不依赖上下文
✅ 每格标注叙事区域（冷区/暖区/过渡区）以对齐节奏
✅ 每格时长建议 2-4s，总时长控制在 50-100s
✅ 高潮帧（17-19）可加长到 4-5s
❌ 不要在单格提示词中引用其他格的内容
❌ 不要让相邻格的运镜方向冲突（如格01左摇 + 格02右摇）
```

### 与剪辑节奏的联动

25 格拆解表完成后，剪辑时按 [editing-rhythm.md](editing-rhythm.md) 的公式组织：

- **海浪式** 最贴合 25 格默认节奏曲线
- **脉冲式** 适合 16-19 高潮段内部进一步加密
- **子弹时间** 适合 17-19 高潮点本身（慢动作 + 爆发）
